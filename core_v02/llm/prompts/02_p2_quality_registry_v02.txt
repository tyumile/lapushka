Критично:
1) Для КАЖДОГО материала обязателен непустой массив docs.
2) Если поле не найдено, ставь status=needs_extraction, но все равно добавляй запись docs с file_ref и source.
3) file_ref должен ссылаться на конкретный файл качества.
3.1) ОБЯЗАТЕЛЬНОЕ ПОКРЫТИЕ ФАЙЛОВ: в ответе должны быть учтены ВСЕ загруженные файлы из 01_input/02_quality_docs.
3.2) Для КАЖДОГО загруженного quality-файла должна быть минимум одна запись в materials[*].docs[*] с file_ref, равным ТОЧНОМУ пути этого файла.
3.3) Нельзя пропускать файлы как "дубликаты". Если файл нечитабелен/неоднозначен, все равно добавь docs-запись с status=needs_disambiguation или blocked_missing_source.
3.4) Запрещено выдумывать/нормализовать имена вида protocol_1.pdf. Используй только реальные имена загруженных файлов.
3.5) Перед финальным ответом сделай самопроверку: число уникальных file_ref в docs >= числу quality-файлов (повторы file_ref допустимы, если из одного файла извлечено несколько записей).
4) Нельзя возвращать только список материалов без docs.
5) Для КАЖДОЙ записи docs обязательно попытайся извлечь volume:
   - объем в м3, или
   - длину в м, или
   - вес в кг.
   Если не найдено, ставь volume="needs_extraction".
6) project_cipher.value должен быть именно шифром проекта, извлеченным из загруженного проекта/документов (например "..." из титула проекта).
   НЕЛЬЗЯ подставлять project_id, название папки или имя проекта пользователя.
   Если шифр не найден, верни:
   - project_cipher.value = ""
   - project_cipher.status = "needs_extraction"
7) Добавь agent_comments: краткие комментарии/наблюдения, извлеченные из файлов, с обязательным source.
8) Если есть docs со статусом != ok, добавь в agent_comments причину по каждому такому file_ref.
